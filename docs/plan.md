## 一、项目概述

### 1.1 项目背景
基于对 FateMaster.AI 的深度研究，我们计划打造"赛博命理师"—— 一款基于 AI 的东方命理分析网站。MVP 阶段聚焦**网页端**，采用类似 FateMaster 的**表单输入 + 报告输出**模式，快速验证产品价值。

### 1.2 MVP 目标
- 快速上线核心命理分析功能
- 验证用户对"AI 命理"产品的兴趣
- 为后续迭代建立技术基础
- **时间目标**: 2 周内完成 MVP

### 1.3 MVP 定位
| 维度 | MVP 范围 | 后续迭代 |
|------|----------|----------|
| 平台 | 网页端 | 微信小程序、App |
| 交互 | 表单输入 → 报告输出 | 对话式 Agent |
| 功能 | 八字核心功能 | 紫微、塔罗、合盘等 |
| 付费 | 免费体验 | 付费深度分析 |

---

## 二、用户画像

### 2.1 目标用户
**核心用户**: 25-40 岁职场人群
- 对传统文化有兴趣但不迷信
- 面临人生选择时希望获得参考
- 习惯网页端产品
- 愿意为优质内容付费

### 2.2 典型场景
1. **了解自己**: "看看我的命盘是什么样的"
2. **日常参考**: "今天运势怎么样"
3. **人生决策**: "今年适合跳槽/创业吗"
4. **娱乐好奇**: "帮朋友/家人也看看"

### 4.2 核心模块

#### 模块 1: 八字计算引擎
**功能**: 公历日期 → 八字四柱

**技术方案**: 使用开源库 (如 lunar-javascript)

**输入输出**:
```javascript
// 输入
{
  date: "1990-03-15",
  time: "08:00",  // 可选
  gender: "male"
}

// 输出
{
  bazi: {
    year: { gan: "庚", zhi: "午", wuxing: "金火" },
    month: { gan: "己", zhi: "卯", wuxing: "土木" },
    day: { gan: "丙", zhi: "寅", wuxing: "火木" },
    hour: { gan: "壬", zhi: "辰", wuxing: "水土" }
  },
  wuxing: {
    metal: 2,
    wood: 3,
    water: 1,
    fire: 2,
    earth: 2
  },
  dayMaster: "丙火",
  dayMasterStrength: "身旺"
}
```

#### 模块 2: AI 解读生成
**功能**: 八字数据 → 自然语言报告

**技术方案**: 调用 LLM API (如 GPT-4 / Claude)

**Prompt 结构**:
```
你是一位专业的命理分析师，请根据以下八字信息生成分析报告。

## 八字信息
{bazi_data}

## 要求
1. 语言通俗易懂，不故弄玄虚
2. 分析要有依据，说明为什么
3. 给出具体可操作的建议
4. 保持客观，用"倾向于"而非"一定会"
5. 字数控制在 500 字左右

## 输出格式
请按以下结构输出：
- 日主分析（100字）
- 性格特点（100字）
- 事业运势（100字）
- 财运分析（100字）
- 感情婚姻（100字）
```

#### 模块 3: 本地存储
**功能**: 记住用户的生日信息

**技术方案**: localStorage

```javascript
// 存储结构
{
  birthday: "1990-03-15",
  birthTime: "08:00",
  gender: "male",
  savedAt: "2026-03-06"
}
```

```
首页
  │
  ├──点击"开始测算"──→ 八字计算页
  │                      │

  │                      │
```

#### 八字计算页状态
```
初始态 → 填写中 → 提交中(loading) → 结果展示
                      ↓
                   错误提示
```

```
加载中 → 有生日信息 → 显示运势
           ↓
        无生日信息 → 显示输入表单 → 提交 → 显示运势
```

## 六、UI/UX 设计要点

### 6.1 设计风格
- **整体调性**: 现代科技 + 东方元素
- **配色**: 深色主题（神秘感）+ 金色点缀
- **字体**: 思源宋体（标题）+ 系统默认（正文）

### 6.2 响应式适配
- 桌面端 (>1024px): 双栏布局
- 平板 (768-1024px): 单栏宽布局

### 6.3 关键交互
- 表单即时校验
- 提交按钮 loading 状态
- 报告逐段渲染（打字机效果，可选）
