```
└────────────────────────────────────┘
```

#### 交互说明

**邮箱登录/注册（二合一）**:
1. 用户输入邮箱
2. 系统检查邮箱是否已注册
   - 已注册 → 显示密码框，登录
   - 未注册 → 显示密码框 + 确认密码框，注册
3. 注册成功 → 发送验证邮件（可选，V2）

**微信登录（OAuth）**:
1. 点击微信登录
2. 跳转微信授权页（扫码/跳转）
3. 授权成功 → 回调处理
4. 新用户自动创建账号
5. 登录成功 → 跳转来源页

#### 表单验证

| 字段 | 规则 |
|------|------|
| 邮箱 | 必填，邮箱格式 |
| 密码 | 必填，6-20 位，至少包含字母和数字 |
| 确认密码 | 与密码一致（仅注册） |

## 十、风险与备选方案

### 10.1 微信 OAuth

**风险**: 微信开放平台审核周期长

**备选方案**:
- MVP 先上邮箱登录
- 微信登录作为 V1.1 补充
- 或使用服务号网页授权（门槛较低）

### 10.2 支付接入

**风险**: 微信/支付宝商户审核需要时间

**备选方案**:
- 使用聚合支付（如 Stripe、Paddle）
- 或先上线功能，支付后补
- MVP 可考虑"微信转账 + 人工开通"作为临时方案

### 10.3 自动续费

**风险**: 需要签约代扣，资质要求高

**MVP 简化方案**:
- 先做单次购买，不做自动续费
- 到期提醒用户手动续费
- V2 再考虑自动续费
